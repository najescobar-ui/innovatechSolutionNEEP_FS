ENFOQUE: MONO-REPO
------------------
Todos los componentes (frontend, BFF, microservicios y arquetipo Maven)
viven en un único repositorio multi-módulo. Esta decisión se tomó para:
  - Mantener un único `pom.xml` padre que centraliza versiones del stack.
  - Garantizar atomicidad en cambios cross-componente (ej. agregar un
    endpoint nuevo toca BFF + microservicio en el mismo commit).
  - Simplificar el `docker compose up` para presentación: un solo `git clone`
    y un solo comando levanta toda la infraestructura.

Las rutas indicadas debajo apuntan a las subcarpetas dentro del mismo repo
en la rama `main`.

 COMPONENTES INDIVIDUALES (rutas dentro del mono-repo)
===========================================================================

------------------------------------------------------------
 1. FRONTEND  (Componente NPM)
------------------------------------------------------------
Ruta:       /frontend/
Enlace:     https://github.com/najescobar-ui/innovatechSolutionNEEP_FS/tree/main/frontend
README:     https://github.com/najescobar-ui/innovatechSolutionNEEP_FS/blob/main/frontend/README.md

Descripción:
  Aplicación React 18.3 + Vite + TypeScript empaquetada con el estándar
  NPM. Tema dark inspirado en Linear/Vercel/Supabase. Se autentica contra
  Keycloak por OAuth2 (PKCE) y consume el backend a través del API Gateway.
  Incluye Dockerfile multi-stage (Node build → Nginx Alpine).

  Vistas implementadas:
    - Login Keycloak (PKCE)
    - Dashboard adaptado por rol (PM / DEV / DIR) con KPIs reales,
      filtro de fechas (rango ≤ 30 días) y gráfico de evolución.
    - Proyectos: tabla densa con CRUD parcial + selección múltiple.
    - Recursos: tabla densa con CRUD parcial + selección múltiple.

------------------------------------------------------------
 2. BFF  (Backend For Frontend)
------------------------------------------------------------
Ruta:       /bff/
Enlace:     https://github.com/najescobar-ui/innovatechSolutionNEEP_FS/tree/main/bff
README:     https://github.com/najescobar-ui/innovatechSolutionNEEP_FS/blob/main/bff/README.md

Descripción:
  Componente Spring Boot que orquesta las llamadas a los microservicios,
  aplica circuit breakers por endpoint (Resilience4j) y construye DTOs

  Patrones aplicados: BFF, Factory Method, Circuit Breaker,
  Service Discovery.

------------------------------------------------------------
 5. MICROSERVICIO — ms-analitica
------------------------------------------------------------
Ruta:       /servicios/ms-analitica/
Enlace:     https://github.com/najescobar-ui/innovatechSolutionNEEP_FS/tree/main/servicios/ms-analitica
README:     https://github.com/najescobar-ui/innovatechSolutionNEEP_FS/blob/main/servicios/ms-analitica/README.md

Descripción:
  Agregador de KPIs cross-organización. Consume ms-proyectos y ms-recursos
  vía REST (Eureka) y persiste snapshots históricos en su BD propia
  (db-analitica) para series temporales: sparklines, deltas y filtro de
  rango de fechas (≤ 30 días). Scheduler Spring que captura snapshots
  cada N minutos.

  Patrones: Repository, Database per Service, Snapshot/Periodic
  Aggregation, Circuit Breaker, Service Discovery.

------------------------------------------------------------
 6. ARQUETIPO MAVEN  (POM padre multi-módulo)
------------------------------------------------------------
Ruta:       /pom.xml  (raíz del repo)
Enlace:     https://github.com/najescobar-ui/innovatechSolutionNEEP_FS/blob/main/pom.xml
Guía:       https://github.com/najescobar-ui/innovatechSolutionNEEP_FS/blob/main/ARQUETIPOS.md

Descripción:
  POM padre `innovatech-parent` que cumple la función de arquetipo Maven
  para todos los componentes backend del proyecto. Centraliza versiones
  del stack (Java 25, Spring Boot 4.0, Spring Cloud 2025.1, Resilience4j
  2.2, Lombok 1.18.44, MapStruct 1.6.3) y configura plugins de build con
  annotation processors. Cualquier microservicio nuevo solo declara
  `<parent>` y obtiene todo el stack heredado.

  La guía ARQUETIPOS.md documenta paso a paso cómo crear un nuevo
  microservicio usando este POM padre como base.
